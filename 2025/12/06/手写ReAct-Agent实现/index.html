手写ReAct_Agent实现

思考过程（Thinking）：

我们被要求“画出流程图”，但这是一个文本聊天界面，无法直接画图。不过我们可以用文字描述流程图的结构，或者提供伪代码的流程描述，或者用Mermaid格式的代码块来生成流程图。用户可能期望看到用Mermaid语言描述的流程图，这样可以在支持Mermaid的平台上渲染。因此，我们可以给出Mermaid格式的流程图。

让我们根据前面的伪代码，设计一个ReAct Agent的流程图。它将展示从输入问题开始，循环直到得到最终答案或达到最大步数。

流程图大致如下：

  • 开始
  • 输入问题
  • 初始化历史记录、步数计数器
  • 循环（步数<最大步数）
    • 构建prompt（包含问题、历史、工具描述）
    • 调用LLM得到响应
    • 解析响应
    • 如果是最终答案 -> 返回答案，结束
    • 如果是行动 -> 提取行动和输入
      • 记录思考
      • 执行工具得到观察
      • 记录行动、输入、观察
      • 步数+1
    • 如果是无法解析 -> 记录错误，步数+1
  • 超过最大步数，返回失败信息

我们可以用Mermaid语法中的flowchart或graph来表示。注意，我们可能需要在每个决策点进行分支。

我将给出一个Mermaid格式的流程图。

以下是 ReAct Agent 核心流程的 Mermaid 流程图，你可以将其复制到支持 Mermaid 的编辑器（如 GitHub、Notion 或在线 Mermaid 工具）中查看：

流程说明：

  1. 开始后，输入问题并初始化历史记录和步数计数器。
  2. 进入循环，检查是否达到最大步数。
  3. 构建提示词（包含问题、历史对话和工具描述）。
  4. 调用大语言模型（LLM）获得响应文本。
  5. 解析响应：
    • 如果包含 Final Answer，则提取并返回最终答案，流程结束。
    • 如果包含 Action 和 Action Input，则提取思考内容（可选）、行动名称和输入参数：
      • 将思考内容加入历史。
      • 检查工具是否存在：若存在则执行工具得到观察结果，否则观察结果为“未知工具”。
      • 将行动、输入和观察结果加入历史。
      • 步数加1，继续循环。
    • 如果无法按预期格式解析，则将错误信息加入历史，步数加1，继续循环。
  6. 如果循环结束仍未得到最终答案，返回超时提示。
