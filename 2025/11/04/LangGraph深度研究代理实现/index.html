LangGraph深度研究代理实现

以下是一个使用 LangGraph 实现的深度研究代理（Deep Research Agent）示例。该代理能够根据用户提出的研究问题，自主规划搜索查询、获取网页内容、总结信息，并最终生成一份综合报告。代码中包含了必要的节点和状态管理，并使用了 Tavily 搜索工具和 OpenAI 的 GPT 模型。

代码说明

  1. 状态定义：ResearchState 包含研究问题、查询列表、URL列表、网页内容、摘要列表和最终报告。contents 和 summaries 使用 operator.add 作为注解，支持在图中累加（尽管本示例是顺序执行，但为扩展保留）。
  2. 节点功能：
    • planner：使用 GPT 生成 3 个搜索查询。
    • searcher：对每个查询调用 Tavily 搜索，收集所有返回的 URL。
    • reader：使用 WebBaseLoader 加载每个 URL 的网页内容，并让 LLM 生成摘要（同时保留原始内容以备后用）。
    • reporter：将所有摘要汇总，让 LLM 生成一份完整的研究报告。
  3. 图结构：简单的线性流程，从规划到搜索、阅读、报告，最后结束。
  4. 运行：给定一个问题，代理将依次执行各节点，最终返回报告。

扩展与优化

  • 增加循环：可以在 reader 后添加一个判断节点，如果信息不足，则重新调用 planner 生成更多查询，形成研究循环。
  • 并行处理：可以使用 LangGraph 的 Send 或并行节点同时处理多个 URL 的读取和摘要。
  • 错误处理：在 reader 中已加入了简单的异常捕获，可根据需要增强。
  • 工具增强：可以集成更多工具，如 Wikipedia、Arxiv 等，丰富信息来源。

这个示例提供了一个基础框架，你可以根据实际需求调整节点逻辑和图结构，构建更强大的深度研究代理。
